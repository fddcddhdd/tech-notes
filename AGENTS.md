# このリポジトリの目的

このリポジトリは、ユーザーが気になった技術的な疑問を、あとから読み返せるHTMLノートとして蓄積するためのものです。

# 基本ルール

- すべて日本語で書く
- 1つの疑問につき、1つのHTMLファイルを作る
- HTML/CSS/画像/SVGだけで構成する
- ビルドツールは使わない
- 既存の `styles/site.css` を使う
- 新しい記事を作ったら `index.html` にリンクを追加する
- 会社情報、個人情報、APIキー、パスワード、未公開情報は絶対に書かない
- 内容に事実関係が含まれる場合は、できるだけ一次情報を確認する
- 記事末尾に「参照」セクションを作る
- 推測と事実を混ぜない
- 不明点がある場合は、不明と明記する

# 記事の構成

新しい記事は以下の構成にする。

1. タイトル
2. ひとことで言うと
3. 背景
4. 詳しい説明
5. 具体例
6. よくある勘違い
7. まとめ
8. 参照

# ファイル名ルール

- `notes/` 配下にHTMLファイルを作る
- ファイル名は英小文字、数字、ハイフンだけにする
- 例: `notes/http-basics.html`

# ユーザーからの指示の扱い

ユーザーが「質問:」または「疑問:」で始めた場合、その内容を技術ノート化する。

例:

質問: HTTPのGETとPOSTの違いがよく分からない

この場合、GETとPOSTの違いを説明するHTMLノートを作成し、`index.html` にリンクを追加する。

# 公開前チェック

以下の内容が含まれる場合は、記事内で断定しすぎないこと。

- 法律
- 税金
- 医療
- 投資
- セキュリティ
- API仕様
- フレームワークのバージョン依存の挙動

これらは必ず参照URLを載せる。
確認できない場合は「要確認」と明記する。
